# TabGen（カレンダー自動生成／転記ツール）
> カレンダーサイトのAEM更新業務を効率化する業務改善ツール

Adobe Experience Manager（AEM）におけるカレンダー更新作業の工数削減、および手作業によるヒューマンエラーの削減を目的に開発した業務改善ツールである。

----

## 開発の背景と課題
- AEMのデータ更新において、システム改修の権限がなく、エディタ機能も形骸化していたため、作業にかなりの時間を要していた。
- また、運用が属人化し、手作業に頼らざるを得ない環境だったため、日付、予定のズレ等のヒューマンエラーが頻発していた。

----

## 解決策
- 既存拡張機能「Tampermonkey」を利用して、カレンダー更新の対象ページ（AEM）上に直接、専用のツールUIを埋め込むスクリプトを開発。
- データ入力から対象ページへの転記までを別ウィンドウに移ることなく完結させ、業務効率化を実現。

----

## Tampermonkeyを利用した理由
今回のツール開発にあたり、以下の理由からTampermonkeyを採用した。

- **導入のハードルとセキュリティの問題**
  RPA、Power Automateツールや個人アプリの導入は、セキュリティポリシーに壁があり、即座の課題解決が困難だった。
  すでに導入されている拡張機能「Tampermonkey」を利用する事で、安全かつ迅速にツール開発を実現。
- **運用リスクの回避**
  RPAなどを個人で構築してしまうと、将来のチーム変更の際に誰もメンテナンスが出来ず、手動作業に戻ってしまうリスクがあった。
  また、RPAライセンスの取得や、専用ツールを一から設計・開発・テストするアプローチは、コストパフォーマンスの観点から合理的ではないと判断した。

----

## 主な機能
- **入力・自動生成機能**
  数字＋α（am、pm、休）を入力して実行ボタンを押すだけで、カレンダーを自動生成。（記入例:「10am」や「15休」など）。
- **HTML形式でのクリップボード自動保存**
  Blobを活用し、生成されたカレンダーデータをHTML形式に変換して自動でクリップボードに保存。
- **AEMへのワンタッチ入稿**
  カレンダー生成後は、AEMのコンポーネント上で「Ctrl＋V」を押してペーストするだけのシンプルな運用フローを実現。

----

## 技術スタック

### フロントエンド
- **言語**
  JavaScript / TypeScript
- **ライブラリ**
  React
- **状態管理**
  useState
- **デザイン**
  インラインスタイル / CSS Transitions

### データ処理のロジック
- **日付操作**
  標準のDate APIを使用。
- **テキスト抽出**
  正規表現による入力データの抽出・全角半角変換。

### 非同期処理・ブラウザAPI
- **非同期処理**
  Async / Await による非同期データ処理。
- **クリップボード・データ連携**
  Clipboard API / Blob APIを使用し、文字列をHTML形式に変換してクリップボードへ保存する機能を実装。

----

## 導入効果
- **作業工数の削減**
  今までのAEM更新にかかっていた時間を、1時間から10分と**約83%削減**。
- **運用時の課題解決**
  本ツールの操作マニュアルを共通化する事で、誰でも対応可能な運用体制を実現。
